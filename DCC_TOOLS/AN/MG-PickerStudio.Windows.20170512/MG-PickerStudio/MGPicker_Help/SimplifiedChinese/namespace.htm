▪ 设置当前picker的namespace：

Namespace是当前Picker的命令空间，即前缀名。改变这个前缀名，使你可以将Picker在所有其它命名类似的角色上使用。

-在设计师模式下设置namespace的两种方式:

1.选择reference角色的任一部分，按Ctrl+Alt+Shift+S快捷键，打开namespace设置窗口并设置。

2.选择reference角色的任一部分，点击右上角的更多功能按钮 /设置Namespace..

 

▪Sub-Namespace(次命名空间):

大部分类型的picker对象都支持一个叫sub-namespace(次命名空间)的属性，(具体是这些类型: 选择按钮，命令按钮，属性按钮，滑条)

sub-namespace是对picker所用namespace的追加。比如您的picker namespace被设置为"myCharacter", 有一个选择按钮的sub-namespace设置为"bodyRig:chestRig", 这个选择按钮含有一个成员控制器叫"chest_ctl",那么这个控制器最终将被解释成：

 myCharacter:bodyRig:chestRig:chest_ctl;

这意味着在动画师模式下，您点击这个选择按钮，它将试图在您的maya场景里选择一个叫"myCharacter:bodyRig:chestRig:chest_ctl"的物体.

这个属性主要是用来支持基于reference的rigging架构，因为在这种rig里，可能已经包含reference。

这样所有这些不同的namespace的物件仍然可以被设计在一个picker文件/节点里使用。
