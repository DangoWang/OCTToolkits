▪多个加载相关Picker的方法:

 

1.默认情况下，所有场景内的Picker节点在读取maya档案后将会被自动加载。
这个自动加载与否的选项可以通过选择更多功能按钮 /配置..，在弹出的配置对话框里设置。
 

2.选择Reference角色的任意一部分， ALT+鼠标左键双击Picker标签的空白处或如果当前没有Picker打开就直接双击视图区，来搜索并自动打开与当前角色相关的Picker。
也可以直接点击右上角的查找&加载Picker按钮 .
也可以点击右上角的更多功能按钮 / 查找&加载Picker.
搜索的优先级及搜索路径可以在配置对话框里设置。
 

3.鼠标左键双击Picker标签的空白处或如果当前没有Picker打开就直接双击视图区，来打开文件对话框，您可以手动拾取一个外部的.mgpkr的picker文件。
 

4.SHIFT+鼠标左键双击Picker标签的空白处或如果当前没有Picker打开就直接双击视图区，来加载场景里所有的Picker节点(如果他们被设置不被自动加载)。
也可以点击右上角的更多功能按钮 / 加载场景内所有Picker节点。
 

5.拖拽一个 .mgpkr文件夹Picker视图区域并松开来打开那个picker文件。

 

 Picker被加载后，你可能仍然需要设置下它的namespace（命名空间）以便它可以正常工作。

 在动画师模式下，可以查看顶上的namespace栏，看上面显示的namespace是否跟你reference角色的namespace是否匹配。

 点此了解 更多namespace的信息。

 

▪MG-Picker工作室搜索三个基本路径来取得相关Picker:

▪在当前Maya场景/ Reference场景内部的Picker节点。
这些Picker节点采用Maya内置节点类型 "subdivSurfaceVarGroup"，并通常有一个"_MGPIKR"作为名字后缀。
Picker节点采用的图片一般存储在Maya场景目录的pickerimages子目录。
 

▪当前或Reference场景目录里的pickers文件夹。
你可以将Picker数据及图片发布在相关场景所在目录里的pickers目录里。这个目录也将被自动搜索。
 

▪Maya的用户程序目录 / MG_PickerData / 当前Picker工程目录。
在Windows操作系统，这个目录是我的文档/maya/MG_PickerData/. 在里面的当前Picker工程里的Picker数据将被搜索。
 

在配置窗口里，你还可以添加其它的自定义搜索路径。
