共享资产的流程：

1.由系统工程师，在项目服务器上生一个目录，这个目录只有将来需要维护全局库的人才有写的权限，其它使用者都只有读的权限。

2.负责建立和维护全局库的人，（我们暂且叫他Asset/Layout leader）建立他本地的资产库。（也可以直接在全局状态下直接建立全局库，不过这样少了备份）

3.Asset/Layout leader在工具里上传他建立的本地库到全局库。

4.Asset/Layout leader在工具里上传他本地maya用户目录/MGTools_GlobalData/MG_AssetLib/相应库工程文件夹/库工程名称.assetproj。

5.Layout人员在工具里，用打开工程方式，打开这个被共享的assetproj文件。

6.这样就可以开始使用了。Layout人员可以用本地库建立和使用自己的库，也可以使用被共享的全局库。

7.当全局库有更新，每个Layout人员都可以立即使用到。

 

上传/下载资产库：

在工具菜单栏的操作菜单/ 上传/下载 菜单里不同级别数据的上传下载功能。

在内容区点右键，向上划，可以访问到上传/下载的marking menu.

上传/下载当前类别勾选的项：上传或下载当前类别被打上勾的资产项。 如果全局或本地的目标目录已有相应的类别，则会合并，避免重复。

上传/下载所有类别勾选的项：：上传或下载所有类别被打上勾的资产项。如果全局或本地的目标目录已有相应的类别，则会合并，避免重复。

上传/下载当前类别：上传或下载当前浏览类别所有的资产项。会确认，让您选择下面几种上传/下载方式：

▪更新类别：当目标库的目标类别已存在， 将作合并

▪替换类别：当目标库的目标类别已存在， 会直接覆盖掉旧文件。

上传/下载所有类别：上传或下载当前资产库所有浏览类别所有的资产项。会确认，让您选择下面几种上传/下载方式：

▪更新所有类别：当目标库的目标类别已存在， 将作合并

▪替换所有类别：当目标库的目标类别已存在， 会直接覆盖掉旧文件。

上传/下载库工程文件：上传或下载当前资产库的.assetproj文件。

上传/下载所有预览图：上传或下载当前资产库目录下的Preview_Images目录。
