使用Pose库：

主界面下方，是库的应用选项。
     

点击模式：
         库管理：这一模式用来管理库，它可以支持ctrl及shift键多选，用来编辑，上传下载，删除库等。

         简单应用：这一模式下，直接单击相应的pose项按钮，来应用pose到当前场景所选。

         高级应用：这一模式下，要先在内容区选择相应的pose项，然后调整下方的应用选项，再点"高级应用"按钮。

 提示：下面的选项在处于简单应用，高级应用模式时才会出现：  

[ Ctrl点击：]：这一栏输入的百分比，决定了每次按Ctrl键同时点击项按钮或者高级模式下的“高级应用”按钮时，应用Pose的程度百分比。

         没有ctrl键的点击，即是100%应用。

命名空间使用：

▪所选物件命名空间： 将把当前场景所选物件的命名空间(namespace)抓出来，替换pose里的控制器的命名空间，得出的映射过的名称，即是要应用pose的对象。
简单说来，假设导出pose时选的是charA:body, charA:hand, 导入时你选的是charB:foot, 则会被导入pose的是charB:body,charB:hand.

▪Pose命名空间：原原本本找到pose被创建时涉及到的控制器，作为应用pose的对象。

▪自定义： 选上以后，右边的输入框可用。你可以手动输入命名空间来作映射。也可以在场景里选择一个对象，用"<选择"按钮将其namespace放进去。

提示：目前MG-Pose动作库，在抓取要应用pose或动作的目标对象时，是通过抓名字的方式而不是直接应用于场景所选择的物件。
           所以如果pose库里涉及到的控制器就是你要应用的对象，你甚至可以不选任何对象就直接来应用。
 

提示：下面的选项在处于高级应用模式时才会出现：

  筛选对象： 决定经过上面规则映射出来的应用对象，是否还要再筛选。

▪不筛选： 直接按上面的规则，抓目标对象来应用Pose及动作。

▪只有所选物件：按上面的规则映射出来的应用对象，只有当前在场景里被选择的，才会被应用Pose.

▪忽略所选物件： 按上面的规则映射出来的应用对象，如果在场景中被选择，就不会被应用Pose.

  筛选属性： 在经过上面规则映射并筛选过后的应用对象，再对其被应用的属性进行筛选。

▪不筛选： 应用于对象上的所有属性。（当然限于有记录pose的属性。）

▪只有所选属性：应用对象在通道栏上所选的属性，才会被应用Pose.

▪忽略所选属性： 应用对象在通道栏上所选的属性，会被忽略不被应用Pose.

  通道栏按钮： 点击来切换，筛选属性是基于Maya通道栏还是MG通道栏所选属性。

  应用选项：

▪到当前帧：pose应用于当前帧。

▪到Pose创建帧：pose创建时在哪一帧，应用时就应用在哪一帧。

▪应用于世界坐标：pose在创建时，只要是变形节点，都会被导出世界坐标的变形信息。
所以应用时，可以以世界坐标的方式来应用这些位移，旋转。
这样即使变形节点的父节点在不同位置，被应用pose的变形节点也会在世界空间的同一位置上。

  高级应用按钮：在高级模式下，你必须先在内容区选择一个项按钮，然后点这个按钮来实际应用。

  滑条：在高级应用按钮下面，你可以直接拖动滑条，来实时地以0%~100%间应用pose.
