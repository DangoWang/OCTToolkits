建立库工程：

▪点击界面左上角按钮/新建库工程..

▪点击菜单栏库工程菜单/新建库工程..

▪如果当前本地没有任何库工程，直接点击层级显示栏上的新建库工程..按钮

 

新建库工程对话框：

在第一行输入库工程的名称，建议直接用你制作的动画项目的名称，不过注意，请不要太特殊的符号，一定要用字母数字和下划线。

因为将来需要实际建立一个该名称的目录。

 

库工程描述： 输入一些描述文件，说明该工程状况，也可以不输入。

全局目录设置：这一栏很重要。

         对于Pose动作库来讲，库都分成本地库和全局库。本地库是给用户自己使用的，放置自行建立的库。

                     全局库，则一般由动画leader来建立，放在大家都可以访问的服务器某路径下。在整个动画流程里共享。

                     本地库的目录是写死的，它一定是在 当前Maya用户目录/MGTools_GlobalData/MG_PoseAnimLib/下面，并以各库工程名来组织。              

                     但相对应的，全局库，可以设为默认的，即与本地库一样的（即是没有全局库），也可以指向服务器上的某个目录。

                     这一栏要设置的，即是全局库的目录指定。默认情况下，是让全局库跟本地库路径一样，即是没有全局库概念。

 

          全局根目录： 即指定一个全局pose库及动作库共用的根目录。                                

                         如果选择 "不需要全局库： 即没有全局库，即全局库跟本地库一样的。                                

                         也可以选择手动指定一个专门针对这个项目的服务器上的一个即有目录。这个选择比较常用。

                         如果路径为空，则还是让全局库目录跟本地目录一样。即是没有全局库。

          全局Pose库设置：在上面设置完 全局库根目录情况下，再进一步指向其下的全局Pose库路径。

                         如果选择 "使用 [全局库工程目录 ]/ PoseLib: 则全局pose库路径是全局库工程目录下的PoseLib目录。这个选择比较常用。

                         也可以选择手动指定一个专门针对这个项目的服务器上的一个即有目录作为pose库目录。这样有可能Pose库跟动作库会分开在不同目录。

                         如果路径为空，则等同于第一个选择。

         全局动作库设置：在上面设置完全局根目录情况下，再进一步指向其下的全局动作库路径。

                         如果选择 "使用 [全局库工程目录 ]/ AnimLib: 则全局pose库路径是全局库工程目录下的AnimLib目录。这个选择比较常用。

                         也可以选择手动指定一个专门针对这个项目的服务器上的一个即有目录作为动作库目录。这样有可能动作库跟Pose库会分开在不同目录。

                         如果路径为空，则等同于第一个选择。

         这一栏解释起来很费劲，但实际上是理解了就很简单。

创建按钮：创建工程并在界面上打开。

取消按钮：取消工程创建。

  注意：所有库工程创建完后，其库工程 文件.libproj，一定是创建在本地根目录上，即当前Maya用户目录/MGTools_GlobalData/MG_PoseAnimLib/库工程名 下面。
