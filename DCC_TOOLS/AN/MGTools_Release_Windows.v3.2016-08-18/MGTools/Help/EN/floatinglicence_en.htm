Node-Locked Licence vs Floating Licence:

Node-Locked Licence: Bound to machine, not transferable, need to pay for future brand new features, but the license has no time limitation as long as you are using the same machine.
Floating licence: Free to login and use MGTools at any machine, no update fee, but you need an internet access and need to pay recursively.

                Tips : If the internet is not totally available, at least you should be able to visit IPs that can be retrieved by "ping ls1.twincodes.com"  and "ping ls2.twincodes.com".

 

Distributing MGTools under Floating License: You still follow the 1~6 steps to deploy MGTools in your pipeline.

1.Visit http://twincodes.com to download the MG-LicenceConnector & MG-LicenceProtector standalone applications (Available in one package).

2.Put the programs in server, those programs are portable, just unzip them to server.

3.Go to MGTools server program folder, go to MGTools/MGTools_ServerData/MG_LicenceConnector.txt.
Input the directory path at the right side of "ConnectorDirectory=", both relative and absolute path are supported.
If this is not set, then each user has to right click the first icon of MGTools, go to "Floating Licence/login..", click the option button, and in the dialog, manually pick that executable for one time.

4.After that, each time you go to "Floating Licence/login..", the MG-LicenceConnector will be launched.

5.When MGTools is loaded:
If a certain machine has node-locked licence, MGTools will use that licence first.
If not, it will test if the position of MG-LicenceConnector is known. If it is , it will launch the program and try to connect to it.
Tips : In Windows & Macos, you could close / minimize the tool to hide it, make the system tray icon visible.
In Linux, since the desktop environment could be various kinds, sometime tray icon are not supported well by Qt, so by default it won't hide the UI and show tray icon by minimizing or closing it.
If the system tray icon does show up correctly, you could turn on Misc/Hide On Minimize in the connector menu.

6.When you have already logged in licence in MG-LicenceConnector, but MGTools still remains unregistered, just go to "Floating Licence/login.." again to remind it.
 

To avoid input username / password each time you use MG-LicenceConnector:

Use the "Remember me" option in login ui.

Another way is to store username and password into a encrypted file generated by MG-LicenceProtector. Checkout info below.

 

If you are software manager and wanna protect the user name and password as a studio property:

1.Use the MG-LicenceProtector. It is a standalone program shipped with MG-LicenceConnector.

2.Load up the MG-LicenceProtector, switch to product to MGTools,  input the user name and password.

3.Setup some filter:
Global IP filter: Input the global IPs your studio use, so only computers within your studio could connect to floating licence.
User-Code filter: Limit which machines could connect to floating licence.
Date filter: If you are freelancer and only work in studio for short term, use the date limitation, and only during that date range the licence could be connected.

4.Save it as .mgtoolslic file, the file should be saved to "licences" folder beside MG-LicenceConnector in order it be picked up automatically.

5.The file is encrypted, to edit it further using MG-LicenceProtector, you have to input password every time.

6.If the license file is placed correctly, it will be used every time MG-LicenceConnector launched, if not, go to "Log in/out / Login all server licence(s)" menu in MG-LicenceConnector.
