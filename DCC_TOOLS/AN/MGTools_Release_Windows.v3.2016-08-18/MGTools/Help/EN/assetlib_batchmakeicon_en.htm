Batch Making Asset Preview Images:

Check on some assets that you want to make their preview images, use one of the following entries to open up the dialog:

▪Click on the Batch make preview image icon in UI.

▪Asset Item/Batch Making Preview Images..

Because the tool will eventually open different scene without any prompt, so before the dilog show up, it will first warning you if your scene is in unsaved state.

 

Batch Making Asset Preview Image Dialog:

The list left shows the checked asset items.
At the left side of each asset item, there is a state icon:

A clock icon means it is pending, a green right arrow means it is current WIP.

A green tick means it is finished, a red excalmatory mark means some errors happened.

The right area of the dialog is the MG-Snapshot component.
Log area:  The operation logging. Usually batch making images for a big asset project is pretty time consuming,

you can go away to get some coffee and get back to check out the log :)

 
Before Snapshot:

▪Stop So I Could Choose Camera Bookmark: Will stop after each asset is opened, so user could tweak the snapshot cameras and continue.

▪Use Default Camera Bookmark: Apply the default "~[4]default" camera bookmark after each asset is opened and continue snapshot.

▪Use Camera Bookmark: Select this mode, and manually pick a camera at the right side, then after each asset is opened, tool will apply this camera bookmark and continue.

File Mode:  Decide how the new preview images fit in the old images:

▪Append To Existed Images: Will carefully not get a duplicate name, so new preview images are append , old image not touched.

▪Replace Existed Images: Will first delete old preview images if there is any, then make the new preview images.

Press the Make!! Button to start the batch process, Close button to close and cancel the batch.

  The preview images made by this tool, are always stored at AssetProjectDir/Preview_Images folder.

So if you are at local library, the store path is Maya User Application Dir/MGTools_GlobalData/MG_AssetLib/AssetProjName/Preview_Images.

The images within this folder, will be invoked in related path.

So if you want to share your library among pipeline, better only refer to the preview images that is available to all layout artist.

or you put the images into the Preview_Images folder at your local side, and upload the images with the tool.

Because the images within use relative path, it is still available in global library as long as it is uploaded to the global side Preview_Images folder.
