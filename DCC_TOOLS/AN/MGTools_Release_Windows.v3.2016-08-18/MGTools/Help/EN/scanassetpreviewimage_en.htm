Batch adding preview images:

Press the button on MG-Asset Library UI to open this window:

 

Scan and match options:

Image formats: The target image file type. Unchecked type file won't be in the scan result.

Match Condition: The auto-match condition between a image file and a asset file:

▪DB: Say a asset file called donkey.ma, if "donkey.jpg" is found then they will match.

▪DB*: Say a asset file called donkey.ma, if "donkey_01.jpg" is found then they will match.

▪*DB: Say a asset file called donkey.ma, if "preview_donkey.jpg" is found then they will match.

▪*DB*: Say a asset file called donkey.ma, if "preview_donkey_1.jpg" is found then they will match.

▪Custom: Double clicking on the text field to input a use custom match expression.(Only support * as the wild-card)
If you are using Maya 2010 or later version,just active the filed and do the type in.

Scan subfolder as well: Scan all sub-directories as well.

Scan Depth: Set the search depth, if you don't want to scan the sub folders, use search depth 1.

Only the latest version: If there are image files share the same name ,then use the latest modified file and ignore the elder one.

PickFolder&Scan...: Press this button to pick a folder and begin scanning for image files. The scanning result will be listed out on the bottom of UI.

         It's list manner is quite similar to MG-Asset Library.

         The result area will be divided to two parts ,the left part list out the image items and the right part shows the image.

: Import all checked on image items into MG-Asset Library preview images for matched asset items.

  Don't forget to press the save button to save the imported preview image data.
